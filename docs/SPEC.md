# ТЗ «Генрих» (актуальная версия)

## Роли
- **Сметчик** — готовит сводную смету по проекту.
- **Закупщик** — собирает счета поставщиков под проект.

## Сценарий А: подготовка сводной сметы

1. Сметчик открывает вкладку **«Загрузка смет»** → грузит XLSX сметы Заказчика и XLSX сметы подрядчика.
   - Каждый XLSX парсится в `Estimate` через Claude (`parser_client.py` / `parser_contractor.py`). Сохраняется и распарсенный результат, и raw bytes.
   - Превью таблиц в expander.
   - `title` подменяется на оригинальное имя файла (не tmpXXXX).
2. На вкладке **«Сводная смета»** → кнопка «✨ Сопоставить позиции» запускает `match_estimates`. Получает `list[Match]`, светофор уверенности.
3. Кнопка «🧪 Классифицировать остаток» запускает `classify_contractor_extras` для неприматченных позиций подрядчика → «🧩 компонент» / «🟪 вне сметы».
4. Выбор проекта (создан на вкладке «Счета поставщиков»). Кнопка «🧮 Подобрать цены закупки» → `match_materials`. Подставляет цены закупки в строки материалов.
5. «📥 Скачать Excel» → `build_workbook` со всеми накопленными match'ами + raw bytes исходников + список счетов проекта.

## Сценарий Б: закупщик загружает счета

1. Вкладка **«Счета поставщиков»** → выбор / создание Проекта.
2. Выбор Поставщика из боковой панели (тип = material).
3. **Мультизагрузка** PDF/XLSX счетов одной кучей.
4. Для каждого файла Claude парсит (`parser_supplier_invoice.parse_invoice`) → превью в expander.
5. Одна кнопка «💾 Сохранить все в проект» сохраняет batch.
6. Внизу — список ранее сохранённых счетов проекта (можно удалить).

## Что попадает в итоговый Excel

### Лист «Сводная смета»
- Столбцы (часть скрыта): № / Наименование Заказчика / Ед. / Кол-во / Цена с НДС (скрыта) / Цена без НДС / Сумма с НДС (скрыта) / Сумма без НДС / Цена закупки без НДС / Сумма закупки без НДС / Наименование позиции подрядчика или счёта / Исполнитель или Поставщик + № счёта / Маржа ₽ / Маржа %.
- **Блок «🔧 РАБОТЫ»** → подразделы Заказчика → позиции с матчем подрядчика → субтотал → ИТОГО ПО РАБОТАМ.
- **Блок «📦 МАТЕРИАЛЫ»** → подразделы Заказчика → позиции с матчем поставщика (артикул и счёт виден в нижней строке).
- **Блок «⚠️ ДОП. РАБОТЫ ПОДРЯДЧИКА — ВНЕ СМЕТЫ ЗАКАЗЧИКА»** — позиции kind="extra" из ContractorExtra. Выручка = 0, маржа отрицательная.
- **ВСЕГО ПО ПРОЕКТУ** — суммы трёх блоков формулами, маржа считается на покрытой выручке.

### Маржа считается только по покрытым строкам
Раздел или блок где `SUM_PURCHASE = 0` показывает маржу `"—"`, а не 100%. Формула на каждом субтотале:
```
sum_net_covered = SUMIF(purchase_rng, ">0", net_rng)
margin_abs = sum_net_covered - SUM(purchase_rng)
margin_pct = margin_abs / sum_net_covered
```

### НДС
22% (РФ 2026). У сметы Заказчика цены даны с НДС → делим на 1.22 для расчётов маржи. У подрядчика и в счетах поставщиков обычно без НДС, но в счёте есть флаг `vat_included` — учитываем индивидуально.

### Лист «Аналитика»
- **Метрики проекта** (формулы со ссылкой на «Сводную смету»): сумма у клиента с/без НДС, сумма закупки работ, маржа по работам ₽ и %, кол-во позиций и матчей.
- **Таблица «🔧 Маржа по работам по разделам»** — по разделам Заказчика. Колонка «Статус» = Закрыто / Частично / Нет нижних цен.
- **Таблица «📦 Материалы по категориям»** — категории определяются `classify_material()` (порядок: Кабель → Зарядные → Шкафы → Лотки, иначе «Другое»). Колонки: Заказчик всего / Заказчик покрыто / Закупка / Маржа покрытой % / Покрытие %. Всё формулами со ссылками на «Сводную смету».

### Исходники
Вкладки **«Исходник Заказчика: <имя листа>»** и **«Исходник подрядчика: <имя листа>»** — копии всех листов исходного XLSX с сохранением font, fill, alignment, border, number_format, ширин колонок, высот строк, объединений, freeze panes. Делается `_copy_external_sheets()`.

### Счета поставщиков как вкладки
По вкладке на каждый счёт проекта: шапка (поставщик / №счёта / дата / итого с НДС) + таблица позиций.

## Хранение состояния
- Поставщики, проекты, счета, позиции — Postgres (Neon).
- Распарсенные сметы, raw bytes XLSX, результаты матчей — `st.session_state` (живут пока открыта вкладка).
- Кэш матчей Claude — на диске в `core/.cache/matches/*.json` (на Streamlit Cloud эфемерный — рестарт стирает).

## Что НЕ делаем (сознательно)
- Не делим в коде «Прокладка кабеля» на материал и работу — Claude матчит её и с подрядчиком (работа), и с поставщиком (материал кабеля). Если у позиции есть `price_material` и `price_work`, она появляется в Сводной смете дважды — это не баг.
- Не пытаемся применить к одной строке Заказчика несколько компонентов подрядчика (это в Roadmap).
- Не показываем "included" extras в Excel (только "extra"). Сейчас это видно только в UI как информация для глаз.
