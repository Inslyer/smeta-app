# Гайд для AI-ассистента в проекте «Генрих»

Это короткие, специфичные для проекта правила. Общий стиль работы — в системном промпте Claude Code.

## Окружение
- **macOS, zsh.** Python 3.12 системный (`/Library/Frameworks/Python.framework/Versions/3.12/bin/python3`).
- **venv проекта** в `smeta-app/venv/`. ВНИМАНИЕ: периодически ломается от параллельных `pip install`. Симптом — `pip._vendor.packaging.version.InvalidVersion`. Лечится `rm -rf venv && python3 -m venv venv && venv/bin/pip install -r requirements.txt`.
- **gh CLI** установлен в `~/.local/bin/gh`, авторизован под `Inslyer`. Используется для коммитов и пушей.
- **`smeta-app/` — git-репозиторий**, remote `https://github.com/Inslyer/smeta-app.git`, ветка `main`. **Не пытайся клонировать заново — всё уже настроено.**
- **`.env`** существует и содержит `ANTHROPIC_API_KEY`, `CLAUDE_MODEL`, `DATABASE_URL` (Neon). Не нужно его пересоздавать.

## Стейкхолдеры
- **Юрий** (пользователь) — основатель/руководитель, бизнес-владелец продукта. Технически грамотный, но просит говорить простым языком, без перегруза.
- **ГРАФТИО** — компания, eMobility (зарядные станции). Полный электромонтаж.
- **Заказчик** — клиент ГРАФТИО.
- **Подрядчик (СМР)** — субподрядчик, который делает монтажные работы.
- **Поставщики** — ЭТМ и другие. У ЭТМ цены переменные (каждый счёт — для конкретного проекта), у других могут быть фиксированные прайсы.

## Постгрес
- **Neon**, free-tier, проект `smeta-app`, регион eu-central-1.
- URL живёт в **Streamlit Cloud Secrets** (формат TOML с кавычками!) и в локальном `.env`.
- Если хост в URL `c-4.eu-central-1` — это правильный pooler-endpoint, не редактируй.
- Streamlit Cloud — эфемерная файловая система. **БЕЗ Postgres любые данные пропадают при rebuild**. Никогда не предлагай возвращаться на SQLite в проде.

## Streamlit Cloud
- Деплой `daniila-smeta.streamlit.app` тянет с `Inslyer/smeta-app:main`.
- Push → автоматический rebuild за ~30-60 сек.
- Если изменения не подхватились: Manage app → Reboot. Если совсем плохо — Settings → Clear cache.
- Streamlit secrets — это TOML, не `.env`. **Значения должны быть в кавычках**, `=` с пробелами. Тильяк ниже сразу ронял парсер не раз.
- Secrets автоматически прокидываются в `os.environ` через мост в начале `app_with_ai.py` (цикл по `st.secrets`).

## Что НЕ делать
- **Не предлагать клонировать git или инициализировать репо** — уже настроено.
- **Не предлагать ставить gh** — установлен в `~/.local/bin/gh`.
- **Не лезть читать длинный файл, если можно сначала спросить пользователя** что именно сломалось. Это самая частая ошибка в прошлом чате.
- **Не редактировать `app.py` и `app_min.py`** — это legacy/демо, не используются.
- **Не упоминать «Claude» в UI-строках** — только «Генрих» (см. BRAND.md).
- **Не светить ключи в чат**. Если нужен полный URL/токен — попроси пользователя посмотреть в `.env`.
- **Не предлагать переходить со Streamlit на FastAPI/React** — это локально оптимальный стек, пользователь сам и в одиночку поддерживает проект.

## Что делать
- **Сначала спрашивать**, что именно у пользователя не работает. Скрин > описание словами > предположения.
- **Все изменения коммитить и пушить сразу**. Сообщения коммитов — на английском, императивом, до ~72 символов в заголовке, остальное в body. Не упоминать Claude/Anthropic в коммитах.
- **После каждого пуша** напоминать пользователю обновить страницу и сверить «Версию» в сайдбаре.
- **При работе с Excel** проверять что формулы валидны (опенпикл `wb.save()` не проверяет синтаксис — только Excel ругнётся при открытии).
- **При изменении схемы БД** — добавлять миграцию или явно проговаривать что схема `metadata.create_all()` сама не догонит изменение существующей таблицы. Поле было добавлено только в новой инсталляции — на Neon нужно `ALTER TABLE` руками или сбросить таблицу.
- **Когда матчер кэширует результат** — менять cache_key (хеш меняется при изменении входа) или предлагать пользователю «🗑 Очистить кеш». Иначе изменения промпта не подтянутся.

## Стиль кода
- Type hints везде где не больно. `Optional[X]` через `Optional` из typing — не `X | None` (для совместимости со старыми Streamlit и читаемости с dataclass).
- Dataclass'ы вместо TypedDict.
- SQLAlchemy Core, **не ORM**. `select()`, `insert()`, `text()`. Возвращаем dict'ы (через `_rows_to_dicts`) — обратная совместимость с предыдущими `sqlite3.Row`.
- Streamlit-кэширование (`@st.cache_data`) пока не используется. Если предложишь — обоснуй, потому что session_state с явной очисткой обычно понятнее.
- Не плодить файлы документации без явной просьбы. Если просят «добавь X» — добавляй в существующий MD.
