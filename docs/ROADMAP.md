# Roadmap «Генриха»

Сгруппировано по приоритету. Хочется или нет — отдельный фильтр.

## Сейчас блокирует (см. KNOWN_ISSUES.md)
- Сметы не загружаются (на Streamlit Cloud, сборка `3f198ee`). До починки никакие фичи делать нельзя.

## P1 — несложно и нужно
- **Маржа покрытой части для работ** в Аналитике, как уже сделано для материалов. Сейчас «Маржа по работам по разделам» считает по всем строкам, в т.ч. без матча подрядчика. Должна быть колонка «Заказчик покрыто (без НДС)» как у материалов.
- **Extras в Аналитику** — отдельной строкой «Доп. работы подрядчика (вне сметы)» с суммой закупки. Сейчас она в Сводной, но в аналитике не учитывается.
- **Скрытие компонентов (included extras) в Сводной смете** или хотя бы пометка ⓘ в строке родителя.
- **«Применить компоненты к закупке родителя»** — если 3 строки подрядчика классифицированы как included с одним и тем же `parent_client_idx`, их суммы добавляются в SUM_PURCHASE строки родителя.
- **Кнопка повторного парсинга счёта** — если пользователь видит ошибку в распарсенном счёте, должна быть возможность отправить тот же файл в Claude ещё раз с подсказкой.

## P2 — расширения
- **История проектов** — после генерации Excel запоминать какой проект и когда был собран, давать список.
- **Сравнение версий** — две сметы Заказчика по одному проекту, дельта.
- **Эталонная база цен поставщиков** — для фикс-прайс поставщиков загружать прайс одним документом, дальше материалы матчатся к нему без счёта.
- **Авторизация** — Streamlit-Auth (Google/Microsoft) когда инструментом начнут пользоваться несколько сотрудников.

## P3 — дальняя перспектива
- **Экспорт в 1С** или другую учётную систему.
- **Уведомления** Slack/Telegram о готовых сметах.
- **Мобильный режим** (Streamlit сжимается, но не приспособлен).
- **Multi-tenant** если решит превратить в SaaS.

## Эпики, которые мы обсуждали и пока отложили
- **Multi-supplier match для одной позиции материала** — иногда один артикул есть у двух поставщиков по разным ценам. Сейчас матчер берёт что-то одно. Идея — показывать топ-2 с подсветкой более выгодного.
- **Конкуренция подрядчиков** — несколько XLSX подрядчиков на один проект, выбор лучшего по строкам.
