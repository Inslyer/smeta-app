# Открытые баги в Генрихе на момент перехода

## 🚨 Блокирующий: сметы не загружаются

**Сборка:** `3f198ee` (commit «ux: build version + clear-session-cache button in sidebar»).

**Симптом:** пользователь подгружает XLSX сметы Заказчика и/или подрядчика, ничего не происходит / падает / нет прогресса.

**Что неизвестно** (нужно спросить в первую очередь):
1. Точный текст ошибки на экране — есть ли красная плашка `st.error`?
2. Стэктрейс в Streamlit Cloud → Manage app → Logs?
3. Грузится ли вкладка вообще, или падает сразу при рендере?
4. Какая «Версия:» в сайдбаре?
5. Какие файлы пробовал — обычные .xlsx или какие-то особые?

**Гипотезы (от вероятной к менее вероятной):**

1. **Кэш Streamlit Cloud не подтянул новый код.** Версия в сайдбаре `3f198ee` — но в Cloud может быть старая. Manage app → Reboot.
2. **Ошибка в _build_version()**. Запускает `subprocess.check_output(["git", ...])`. На Streamlit Cloud `.git` может быть недоступен или git не установлен. У меня в коде `try/except` ловит — должно работать. Но возможно отлавливается не та exception, и крэшится `Path(__file__).stat().st_mtime`. Стоит проверить логи.
3. **Поломка кэша секретов.** На прошлой итерации добавил блок `_secrets_loaded` / `_secrets_error` — мог сломать порядок инициализации. Проверить что `db.init_db()` вообще выполняется.
4. **БД-схема разъехалась.** Если когда-то с локального SQLite кто-то накатывал старую структуру на Neon — какая-то таблица может оказаться без новой колонки. `metadata.create_all` не делает `ALTER`.
5. **Streamlit ругается на CSS** — у нас длинный inline `<style>`, может что-то развалилось.

**Первое действие:** спросить у пользователя скрин или текст ошибки. Не лезть читать `app_with_ai.py` строки 1-100 наугад.

## Прочее не критичное
- **Брокен venv** — пересоздавать при ошибках pip.
- **PNG/SVG логотипа** растягиваются по-разному в Sidebar (40px) vs Header (56px). Можно `preserveAspectRatio`.
- **Дисковый кэш матчей** на Streamlit Cloud не переживает rebuild — пользователь платит за повторы. Перенести кэш в Postgres-таблицу `match_cache` — кандидат на P1.
